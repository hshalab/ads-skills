# Google Ads - API Reference

Scripts that let Claude Code manage a Google Ads account programmatically, using the official `google-ads` Python library.

Location: `.claude/skills/google-ads/scripts/`. Run scripts from that directory.

## Setup

### 1. Google Cloud project
- Enable the **Google Ads API** (APIs & Services -> Library).
- Create an **OAuth 2.0 Client ID** of type **Desktop app** (APIs & Services -> Credentials). Copy the Client ID and Client Secret.

### 2. Developer token
- In Google Ads: Tools & Settings -> API Center. Copy your **Developer Token**.
- A test/Explorer token works on your own accounts. Apply for Basic access to manage client accounts.

### 3. Credentials
Put these in the repo-root `.env` (run `/onboarding` or copy `.env.example`):

```
GOOGLE_ADS_DEVELOPER_TOKEN=...
GOOGLE_ADS_CLIENT_ID=...
GOOGLE_ADS_CLIENT_SECRET=...
GOOGLE_ADS_REFRESH_TOKEN=...        # generated by setup_oauth.py
GOOGLE_ADS_CUSTOMER_ID=1234567890   # 10 digits, no dashes
GOOGLE_ADS_LOGIN_CUSTOMER_ID=...    # your MCC id, only if using a manager account
```

### 4. Refresh token
```bash
cd .claude/skills/google-ads/scripts
pip install google-ads python-dotenv tabulate
python setup_oauth.py
```
This opens a browser, you authorize, and it writes `GOOGLE_ADS_REFRESH_TOKEN` to the root `.env`. Uses a local redirect on port 8080 (the deprecated OOB flow is not used).

### 5. Test
```bash
python account_overview.py
```

## Gotchas
- `use_proto_plus: True` is set in `client.py` - required, or the client throws.
- All new campaigns are created **PAUSED**. Enable explicitly after review.
- Money is in **micros** (1,000,000 micros = $1). The scripts handle conversion in their args.
- Quality Score is 1-10 (higher is cheaper).
- Switch accounts by changing `GOOGLE_ADS_CUSTOMER_ID` in `.env`.

## Scripts

### Reporting
| Script | What it does |
|--------|-------------|
| `account_overview.py` | Account snapshot with optional period comparison (`--compare`) |
| `get_campaign_performance.py` | Campaign metrics, daily breakdown, custom date ranges |
| `search_terms_report.py` | What people actually searched - finds wasted spend and new keywords |
| `get_keyword_performance.py` | Keyword-level metrics with Quality Score |
| `list_campaigns.py` | All campaigns with status and metrics |
| `list_ads.py` | Ads with performance and approval status |

### Management
| Script | What it does |
|--------|-------------|
| `create_campaign.py` | Create a campaign (Search, Display, PMax, etc.). Starts PAUSED |
| `update_campaign.py` | Change status, budget, or name |
| `create_ad_group.py` | Create an ad group with a CPC bid |
| `create_ad.py` | Create a Responsive Search Ad (validates headline/description limits) |
| `add_keywords.py` | Add positive or negative keywords (broad/phrase/exact) |

### Internal
| File | What it does |
|------|-------------|
| `client.py` | Shared auth wrapper - all scripts import `get_client()` from here |
| `config.py` | Reads credentials from the root `.env` |
| `setup_oauth.py` | One-time OAuth flow to generate the refresh token |

Each script supports `-h` for its full argument list.

## How Claude Code uses this
- *"Show me account performance for the last 7 days"* -> `python account_overview.py --date-range last_7d --compare`
- *"What search terms are wasting budget?"* -> `python search_terms_report.py --no-conversions --min-clicks 3`
- *"Create a search campaign for brand keywords"* -> `python create_campaign.py --name "Brand - Search" --type SEARCH --budget 50`
- *"Pause the underperformer"* -> `python update_campaign.py --campaign-id <id> --status PAUSED`
